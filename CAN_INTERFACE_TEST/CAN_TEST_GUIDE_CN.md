# CAN 接口验证操作指南

## 1. 这次要验证什么

本工程分两步验证：

1. 上电后先执行 MCU 内部 CAN 回环，检查 CAN 控制器、时钟、发送邮箱、过滤器和接收 FIFO。
2. 回环通过后进入真实 CAN 总线模式，使用 USB-CAN 检查 PD0/PD1、H7 跳帽、TJA1050、CANH/CANL 和板载接口。

只给开发板烧程序、不连接第二个 CAN 节点，只能完成第 1 步。CAN 正常发送需要总线上另一个处于正常模式的节点返回 ACK，所以建议准备一个 USB-CAN 适配器。

## 2. 需要准备

- 当前 STM32F103ZET6 开发板。
- 支持经典 CAN、500 kbit/s 的 USB-CAN 适配器。
- 3 根连接线：CANH、CANL、GND。
- USB-CAN 自带的电脑软件。

USB-CAN 不是普通 USB 转 TTL 串口。购买或使用的设备端子上应能看到 CANH、CANL、GND，最好带可开关的 120 欧终端电阻。

## 3. 先检查板上 H7 跳帽

CAN 电路旁边有一个 2x2 排针 H7，它位于 MCU 信号和 TJA1050 收发器之间。

- H7 的 1-2 要用一只跳帽横向短接。
- H7 的 3-4 要用另一只跳帽横向短接。

也就是需要两只并排的横向跳帽。不能竖向短接 1-3 或 2-4，否则会接错信号。

## 4. CAN 两针插口怎么接

板上丝印 `CAN` 旁边的两针插口在原理图中叫 CN2：

- CN2 第 2 脚是 CANH。
- CN2 第 1 脚是 CANL。

如果插口旁没有直接印 H/L，可以看 PCB 的 1 脚标记或焊盘：方形焊盘通常表示第 1 脚，因此方形焊盘一侧是 CANL，另一侧是 CANH。接线前也可以根据开发板原理图再次确认。

断电后按下面方式连接：

```text
STM32 开发板                         USB-CAN

CAN 插口 CN2-2（CANH）  ----------  CANH
CAN 插口 CN2-1（CANL）  ----------  CANL
开发板任意 GND 针脚       ----------  GND
```

注意：

- 两针 CAN 插口只有 CANH 和 CANL，没有 GND。
- 公共地必须从开发板其他 GND 针脚单独连接到 USB-CAN 的 GND。
- 不要把 USB-CAN 的 VCC、5V 或 3.3V 接到 CANH/CANL。
- 开发板和 USB-CAN 各自正常供电即可。

## 5. 终端电阻

这块开发板的 CANH 与 CANL 之间已经固定安装 R66，阻值为 120 欧。USB-CAN 一端也需要 120 欧终端：

- 如果 USB-CAN 有 `120R` 开关或跳帽，将其打开。
- 如果没有，在 USB-CAN 端的 CANH 与 CANL 之间外接一个 120 欧电阻。

所有设备断电后，用万用表电阻档测 CANH 与 CANL：两端各有一只 120 欧时，应测得约 60 欧。测到约 120 欧通常表示只有一端有终端电阻。

桌面短距离测试时，让 CANH 和 CANL 两根线绞在一起，长度控制在 1 米以内。

## 7. 先看串口内部回环结果

打开之前使用的板载串口，参数设为：

- 波特率：115200。
- 数据位：8。
- 停止位：1。
- 校验：无。
- 流控：无。

复位开发板，应看到：

```text
[can-loopback] result=0 PASS
[can-normal] init=0 bitrate=500000
```

`PASS` 表示 MCU 内部 CAN 功能正常。它不代表外部的 TJA1050 和 CAN 插口已经验证完成，仍需继续以下测试。

## 8. 设置 USB-CAN 软件

不同品牌的软件名称不同，但参数含义相同：

| 参数 | 设置值 |
| --- | --- |
| 类型 | 经典 CAN / CAN 2.0，不是 CAN FD |
| 波特率 | 500 kbit/s |
| 帧格式 | 标准帧，11 位 ID |
| 工作模式 | 正常/主动模式 |
| 监听模式 | 关闭，不能使用只听/静默模式 |
| USB-CAN 端终端电阻 | 120 欧打开 |

设置后启动或打开 CAN 通道。必须先让 USB-CAN 进入正常工作状态，它才会给开发板发出的帧应答 ACK。

## 9. 测试开发板发送

开发板每秒发送一帧：

| 字段 | 预期内容 |
| --- | --- |
| ID | `0x321` |
| 类型 | 标准数据帧 |
| DLC | 8 |
| 第 0-3 字节 | 逐次增加的 32 位计数值 |
| 第 4-7 字节 | `53 54 4D 32` |

USB-CAN 软件接收列表中应每秒出现一帧 `0x321`。串口状态应显示 `tx_result=0`，并且 `tx_ok` 不断增加。板上 LED1（PD13）在每次发送成功后翻转。

如果尚未启动 USB-CAN，串口出现 `tx_result=-1` 是正常的，因为没有第二个节点返回 ACK。

## 10. 测试开发板接收和回复

在 USB-CAN 软件中创建并发送下面这帧：

| 字段 | 设置值 |
| --- | --- |
| ID | `0x123` |
| 类型 | 标准数据帧 |
| DLC | 8 |
| 数据 | `01 02 03 04 05 06 07 08` |

开发板收到后会：

1. 在串口打印收到的 `0x123`。
2. 自动发送 ID 为 `0x124` 的回复。
3. 回复帧的数据原样保持为 `01 02 03 04 05 06 07 08`。

USB-CAN 接收列表应看到：

```text
ID=0x124  DATA=01 02 03 04 05 06 07 08
```

板上 LED2（PD14）在每次收到标准数据帧后翻转。

同时通过“开发板发 `0x321`”和“电脑发 `0x123`、开发板回 `0x124`”两项测试，才能确认板载 CAN 接口双向正常。

## 11. 常见问题

| 现象 | 检查内容 |
| --- | --- |
| 串口完全没输出 | COM 口是否选对，串口是否为 115200 8N1 |
| 内部回环显示 FAIL | 是否烧录了本工程；这通常不是外部接线问题 |
| 一直 `tx_result=-1` | USB-CAN 是否已启动并处于正常模式，波特率是否为 500 kbit/s |
| `tec` 不断增加 | 另一节点没有 ACK，检查 CANH/CANL、公共地、H7 和工作模式 |
| USB-CAN 看不到 `0x321` | 检查 H/L 是否接反、是否误选 CAN FD、终端电阻是否正确 |
| 开发板收不到 `0x123` | 确认发送的是 11 位标准帧而不是 29 位扩展帧 |
| 通信偶尔正常 | 检查两端终端电阻，缩短线缆并将 CANH/CANL 绞合 |
| `bus_off=1` | 修正波特率或接线后复位板子；程序也开启了自动恢复 |

LED3（PD15）点亮表示最近一次心跳没有得到 ACK，或者初始化失败。本板 LED 为低电平点亮。
